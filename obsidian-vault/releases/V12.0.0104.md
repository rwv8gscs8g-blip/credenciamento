# Release Notes — V12.0.0104

**Data**: 2026-04-09  
**Autor**: Agente IA (Cursor)  
**Tipo**: fix(infra)

---

## Resumo da Iteracao

Reverte a estrategia das iteracoes V12.0.0094–V12.0.0103 que **desqualificavam** chamadas (`Modulo.Procedimento`) em massa. No Excel para Windows, o padrao **qualificado** e o esperado e corresponde ao codigo estavel da **V12.0.0093** (ultima referencia de compilacao informada pelo usuario). Mantem-se **CNAE**: `Util_CNAE`, `Auto_Open` (best-effort), enums `EVT_CNAE_IMPORT` / `ENT_INFRA`, bateria **sem apagar ATIVIDADES** no reset.

---

## Problema (causa raiz)

Varias IAs assumiram que `NomeModulo.Funcao` causava erro de compilacao e substituiram por chamadas globais. Isso:

- gerou **ambiguidade** (`Inserir`, `BuscarPorId`, etc. em varios Repos);
- espalhou correcoes parciais arquivo a arquivo;
- afastou o projeto do comportamento da **0093**.

---

## Mudancas

### 1. Restaurar qualificacao padrao (estilo V12.0.0093)
- **Menu_Principal**: `ErrorBoundary.*`, `Util_Conversao.*`, `Funcoes.*`
- **Preencher**: `Util_Conversao.ToDouble`
- **Util_Planilha**: `Util_Conversao.ToLong`
- **Forms / Repo_Empresa / Credencia_Empresa**: `Funcoes.*`
- **Svc_OS / Svc_Avaliacao / Svc_PreOS / Svc_Rodizio / Credencia_Empresa / Auto_Open**: `Audit_Log.RegistrarEvento`; **Svc_OS**: `Repo_OS.Inserir`, `AppContext.SetOS` / `Invalidate`
- **Svc_Avaliacao**: `Repo_OS.BuscarPorId`

### 2. Logo (Preencher)
- `LogoPrefeitura` restaurado como na 0093, **sem** `.Picture.Visible` (invalido).

### 3. CNAE (mantido)
- `Util_CNAE.bas`, macro `CNAE_ImportarCNAE_Botao`, `CNAE_ImportarBaseServicos`, caminhos CSV.
- `Auto_Open` continua tentativa best-effort com `Audit_Log.RegistrarEvento` em falha.

### 4. Wrappers em Repo_OS
- `InserirOS` / `BuscarOSPorId` **permanecem** (compatibilidade; nao remover API publica).

### 5. Versao
- `Menu_Principal`: **V12.0.0104** / **12104**
