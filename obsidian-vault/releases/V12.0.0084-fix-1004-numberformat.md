# V12.0.0084 — Fix: Erro compilação + Erro 1004 NumberFormat

**Data:** 2026-04-07  
**Compilou:** PENDENTE (aguardando validação no Excel)  
**Runtime:** PENDENTE

## Problema 1: Erro de Compilação

`Central_Testes.bas` linha 118 chamava `Menu_Principal.Menu_RecolherParaBateria`
com early binding. O compilador VBA resolve essa referência em tempo de compilação —
se o formulário Menu_Principal no Excel não expõe esse método (versão antiga importada),
dá "Método ou membro de dados não encontrado" mesmo com `On Error Resume Next`.

## Problema 2: Erro 1004 Runtime NumberFormat

`BA_AtualizarResumo` não verificava `gRegistrarEmPlanilha` e não tinha `On Error Resume Next`.
Quando TESTE_OFICIAL estava protegida, `.NumberFormat` falhava com 1004 sem handler.

## Mudanças (3 arquivos)

### Arquivo 1: vba_export/Central_Testes.bas

`CT_IniciarBateria` (L117-127):
- Substituído `Menu_Principal.Menu_RecolherParaBateria` (early binding)
  por loop `VBA.UserForms` + `frmMP.Hide` (late binding)
- Compilação não depende mais da presença do método no formulário

### Arquivo 2: vba_export/Teste_Bateria_Oficial.bas

`BA_AtualizarResumo`:
- Adicionado `If Not gRegistrarEmPlanilha Then Exit Sub` (guard)
- Envolvido corpo em `On Error Resume Next` / `On Error GoTo 0`
- Se falhar, marca `gRegistrarEmPlanilha = False` (fallback graceful)

### Arquivo 3: vba_export/Util_Planilha.bas

`FormatarComoData` e `FormatarComoMoeda`:
- Envolvidas em `On Error Resume Next` / `On Error GoTo 0`

## Como Validar

1. No Excel: Alt+F11 → VBE
2. Remover e reimportar 3 módulos (da pasta `vba_import/001-modulo/`):
   - `Central_Testes` → reimportar `017-Central_Testes.bas`
   - `Teste_Bateria_Oficial` → reimportar `031-Teste_Bateria_Oficial.bas`
   - `Util_Planilha` → reimportar `006-Util_Planilha.bas`
3. Depurar → Compilar VBAProject → deve compilar sem erro
4. Executar bateria RÁPIDA (Central → Opção 4 → Não)
5. Executar bateria LENTA (Central → Opção 4 → Sim)
6. Verificar: sem erro 1004, TESTE_OFICIAL preenchida, CSV gerado
