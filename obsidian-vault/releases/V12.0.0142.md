# V12.0.0142

## Objetivo

Corrigir lista vazia no formulario Cadastro_Servico e robustez geral da inicializacao.

## Alteracoes

### Preencher.bas

- `PreenchimentoListaAtividade` agora aceita parametro opcional `lstDireto As Object`. Quando fornecido, usa a referencia direta ao ListBox em vez de buscar via `ControleFormulario`. Isso elimina falha silenciosa quando o formulario esta sendo inicializado e pode nao estar acessivel via `VBA.UserForms`.
- Adicionada chamada a `Util_LimparFiltrosAba(wsAtiv)` antes de ler ATIVIDADES, para garantir que filtros residuais nao escondam dados.

### Cadastro_Servico.frm

- Todas as chamadas a `PreenchimentoListaAtividade` agora passam `SV_Lista` como segundo parametro, garantindo que a lista seja populada durante Initialize.
- Afeta: `UserForm_Initialize`, `S_Atividade_Change`, `mTxtBuscaTopo_Change`, `S_Cadastrar_SV_Click`.

### Menu_Principal.frm

- Removida chamada inutil a `PreenchimentoListaAtividade` em `Cad_Servico_Click` (executava antes do formulario Cadastro_Servico existir, sempre retornava sem efeito).

### App_Release.bas

- Versao atualizada para V12.0.0142.

## Diagnostico

A lista no Cadastro_Servico aparecia vazia porque `ControleFormulario("Cadastro_Servico", "SV_Lista")` falhava silenciosamente durante `UserForm_Initialize`. A busca via `VBA.UserForms` + `BuscarControleRecursivo` retornava Nothing, e a funcao saia via `If lst Is Nothing Then Exit Sub`. O erro era engolido pelo handler `On Error GoTo erro_carregamento`.

## Impacto esperado

- Ao clicar "Cadastrar Servico", a lista de CNAEs/atividades aparece populada.
- Filtro de busca no campo "Descricao Atividade" funciona.
- Busca incremental do TextBox topo-direita funciona.
- Nenhuma regressao: chamadas existentes sem o parametro lstDireto continuam funcionando via ControleFormulario.
