# 38. useHBN — Tese da Fagocitose Tecnologica Segura

## Frase-mae

> O **useHBN** nao tenta substituir tecnologias. Ele cria uma camada
> honesta, segura e evolutiva para que humanos e IAs possam
> compreende-las, preserva-las, traduzi-las, testa-las e melhora-las
> em ciclos continuos.

## 1. O conceito central

O **useHBN** nao e uma tecnologia. E um **protocolo vivo de traducao,
documentacao, seguranca e evolucao de tecnologias.**

A "fagocitose" do useHBN nao e consumir para substituir. E **absorver
conhecimento operacional sem apagar a identidade original.**

Uma tecnologia fagocitada continua existindo como:

- linguagem original;
- documentacao estruturada;
- padroes de uso;
- riscos conhecidos;
- conectores;
- exemplos;
- testes;
- traducao para outras camadas;
- modulos reutilizaveis por agentes humanos e IAs.

**Exemplo:** VBA, Pascal, Fortran, COBOL, Delphi, Clipper, Visual
Basic 6, sistemas antigos nao documentados deixam de ser "caixas
pretas" e passam a virar **bibliotecas vivas de conhecimento tecnico.**

## 2. A tese forte — via dupla

### 2.1. IA → entende o passado

As IAs reduzem a friccao para compreender codigo legado, linguagens
antigas, sistemas sem documentacao e regras de negocio escondidas. O
mercado ja esta olhando para isso (Fujitsu 2026 anunciou solucao de
IA para analisar COBOL e gerar documentacao de design via knowledge
graph).

### 2.2. Passado → melhora a IA

A logica antiga e frequentemente mais simples, deterministica,
estavel e resiliente. Padroes do passado viram referencia para
melhorar agentes modernos, que hoje sofrem com excesso de abstracao,
improviso e regressao.

**A fagocitose tem dupla via:** usar IAs para entender o passado E
usar tendencias do passado para simplificar e melhorar a performance
das IAs na operacao.

## 3. Arquitetura logica — 8 camadas

### Camada 1 — Contato

O sistema toca uma tecnologia: repositorio, arquivo, planilha, macro
VBA, codigo legado, documentacao antiga, API, protocolo, linguagem
nova. Saida: **inventario inicial.**

### Camada 2 — Leitura segura

Antes de executar qualquer coisa: leitura estatica, fingerprint da
tecnologia, identificacao de riscos, classificacao de permissoes,
modo sandbox, bloqueio de acoes destrutivas. Saida: **perfil tecnico
+ perfil de risco.**

### Camada 3 — Compreensao

Agentes especializados extraem: intencao, regras de negocio,
estruturas de controle, dependencias, entradas e saidas, estados,
efeitos colaterais, pontos frageis. Saida: **modelo semantico da
tecnologia.**

### Camada 4 — Documentacao viva

O sistema cria documentacao em niveis: humano iniciante, humano
tecnico, agente de IA, mantenedor, auditor, tradutor para outra
tecnologia. Saida: **manual vivo e versionado.**

### Camada 5 — Abstracao

O useHBN separa: o que e regra de negocio, o que e implementacao, o
que e limitacao historica, o que e padrao util, o que e debito
tecnico, o que pode virar protocolo. Saida: **padrao reutilizavel.**

### Camada 6 — Traducao

A tecnologia passa a ser acessivel por: comandos naturais,
conectores, agentes, testes, API interna, mapas entre linguagens.
Saida: **ponte universal.**

### Camada 7 — Validacao

Nada entra como verdade sem validacao: testes automatizados, revisao
humana, comparacao antes/depois, execucao controlada,
rastreabilidade, rollback. Saida: **conhecimento confiavel.**

### Camada 8 — Evolucao distribuida

Usuarios podem contribuir com ciclos de tokens: processar
documentacao, melhorar testes, revisar codigo, comparar linguagens,
detectar regressoes, gerar exemplos, enriquecer conectores. Saida:
**microevolucoes continuas.**

## 4. Modulo inicial — HBN Phagocyte: VBA Edition

VBA e o primeiro alvo porque costuma ter:

- regras de negocio escondidas
- automacoes criticas
- pouca documentacao
- dependencia de usuarios humanos
- logica procedural clara
- alto risco de regressao

O caso real ja em curso e o **Credenciamento V12.0.0203**, que ja
produziu lessons L1-L18 + M1-M7 documentadas em
`usehbn/docs/PHAGOCYTOSIS-VBA-PATTERNS.md`. Esse e o **dataset seed**
do modulo `hbn-phago` na sua primeira encarnacao.

Funcoes minimas do **HBN Phagocyte: VBA Edition**:

- ler modulos VBA
- mapear botoes e eventos
- identificar fluxos
- extrair regras
- gerar documentacao
- criar testes de comportamento
- separar logica de interface
- sugerir refatoracoes seguras
- manter compatibilidade com Excel

## 5. Processamento distribuido de tokens

A ideia e parecida com computacao voluntaria (BOINC), mas aplicada a
**tarefas cognitivas**.

A unidade nao e CPU/GPU. E **unidade cognitiva de contribuicao:**

- "documente esta funcao";
- "compare esta macro com este teste";
- "verifique se esta explicacao esta correta";
- "traduza este padrao VBA para pseudocodigo";
- "extraia riscos deste modulo";
- "melhore este conector".

Forma uma especie de **supercomputador humano-IA distribuido**, com
governanca forte e consentimento explicito por sessao (modelo
viral opt-in proposto pelo Codex em 103/103b).

## 6. Riscos principais

| Risco | Descricao | Mitigacao |
|---|---|---|
| **Regressao** | A IA "melhora" algo e quebra comportamento antigo | Camada 7 (validacao) com testes de caracterizacao + rollback obrigatorio |
| **Alucinacao documental** | O sistema documenta algo bonito, mas falso | Camada 7 + revisao humana + comparacao com codigo real |
| **Acoplamento caotico** | Cada tecnologia fagocitada entra de um jeito diferente | Camadas 1-3 com schema canonico + fingerprint padronizado |
| **Execucao perigosa** | Codigo legado pode apagar, sobrescrever, enviar ou alterar dados | Camada 2 (leitura segura, sandbox, bloqueio destrutivo) |
| **Captura por modismo** | O projeto corre atras de frameworks e perde os principios | Frameworks descartaveis; principios permanentes (item 9) |

## 7. Constituicao do useHBN — 10 principios

1. **Preservar antes de transformar.**
2. **Documentar antes de executar.**
3. **Testar antes de refatorar.**
4. **Explicar antes de automatizar.**
5. **Humano no controle por padrao.**
6. **Toda evolucao deve ser reversivel.**
7. **Nenhuma tecnologia fagocitada perde sua identidade.**
8. **O protocolo importa mais que a ferramenta.**
9. **Frameworks sao descartaveis; principios sao permanentes.**
10. **Seguranca e nao regressao tem prioridade sobre velocidade.**

## 10. Sintese das propostas A/B/C/D para o modulo de fagocitose

| Proposta | Storage | Interface | Forte em | Fraco em |
|---|---|---|---|---|
| **A — Antigravity (Markdown/Obsidian)** | Markdown + YAML, vault Obsidian | CLI (Click) | Bootstrap rapido, legibilidade humana | Auditabilidade |
| **B — Antigravity (SQLite/JSON-LD)** | SQLite relacional + JSON-LD | MCP server | Query rica | Friccao alta |
| **C — Codex (Event ledger JSONL)** | JSONL append-only + views Markdown | CLI + MCP read-only | Auditabilidade alta, replay, monotonico | Mais complexo que A |
| **D — Codex (Consent capsules)** | Capsulas autocontidas (lesson.md + evidence.json + redaction-map.json + consent.json) | CLI + site publico | Privacidade forte, network effect | Mais pesado para uso diario |

**Recomendacao distilada (Codex 103c, endossada por Opus 2026-05-02):**
- Implementar **A** para bootstrap rapido (Sprint 0-1)
- Adicionar **campos de capsule de D** desde o inicio (`consent.json`,
  `redaction-map.json`, `license-target.txt`)
- Postpor **B** para apos 50-100 licoes acumuladas
- **C** pode coexistir como camada de event log de A+D mais tarde

## 14. Esteiras Codex em ciclos incrementais

Bastao de execucao (Codex) opera em **esteiras controladas**:

- Esteira E1: Sprint 0 do `hbn-phago` — skeleton CLI Python (Typer +
  uv + pyproject.toml) + ingestao de PHAGOCYTOSIS-VBA-PATTERNS.md
  como seed do empirical base
- Esteira E2: Sprint 1 — read interface (`hbn search`, `hbn show`)
- Esteira E3: Sprint 2 — write interface (lesson template + commit
  helper)
- Esteira E4: Sprint 3 — viral opt-in deposit flow + redaction
- Esteira E5: Wave 11+ — `hbn weekly-review` GitHub Actions

Cada esteira: Codex implementa → Opus valida → Mauricio aprova → merge.

## 15. Frente paralela com Credenciamento

A fagocitose do VBA acontece em paralelo ao desenvolvimento do
Credenciamento. Cada onda do Credenciamento (entregando codigo,
testes, licoes) alimenta o seed do `hbn-phago`. As licoes L1-L18 +
M1-M7 ja documentadas sao o **primeiro corpus empirico** do modulo
de fagocitose.

A relacao e simbiotica:

- Credenciamento → produz licoes empiricas via uso real do VBA
- usehbn → estrutura essas licoes em modulo reutilizavel por outros
  projetos VBA
- Outros projetos VBA → usam o modulo, produzem mais licoes
- Loop fechado de evolucao distribuida
